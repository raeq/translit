# Exceptions

## TranslitError

```python
from translit import TranslitError
```

`TranslitError` inherits from Python's built-in **`ValueError`** — so `except ValueError`
also catches it. translit raises it for invalid normalization/error-mode/platform values,
failed regex compilation, and internal Rust errors.

> **Heads-up — `TranslitError` is not yet the base for *every* error.** Some argument
> checks are currently raised as a plain `ValueError` (mutually-exclusive flags,
> registration limits, an unsupported reverse-transliteration language) or `TypeError`
> (wrong argument type). Unifying these under a `translit`-owned exception hierarchy is
> tracked in [#183](https://github.com/raeq/translit/issues/183). `except ValueError`
> covers `TranslitError` but **not** the wrong-argument-type `TypeError`s — for full
> coverage today, catch `(ValueError, TypeError)`.

Raised when:

- An invalid normalization form is specified (e.g., `form="INVALID"`)
- An invalid error mode is specified (e.g., `errors="unknown"`)
- An invalid platform is specified (e.g., `platform="bsd"`)
- A regex pattern fails to compile (in `slugify(regex_pattern=...)`)
- Any other internal error occurs in the Rust layer

### Example

```python
from translit import normalize, TranslitError

try:
    normalize("text", form="INVALID")
except TranslitError as e:
    print(f"translit error: {e}")
```

### Catching errors

Because `TranslitError` subclasses `ValueError`, you can catch it specifically, as a `ValueError`, or via a broader `Exception` handler:

```python
try:
    result = slugify(text, regex_pattern="[invalid")
except TranslitError:
    # Handle translit-specific errors
    pass
except Exception:
    # Handle any error
    pass
```

### Error messages

Error messages describe the specific issue. Most are generated by the Rust core; the
`errors=` and `form=` checks are currently *also* validated in the Python wrapper's ASCII
fast-path, with identical text (that duplication is tracked in
[#185](https://github.com/raeq/translit/issues/185)):

```
form must be 'NFC', 'NFD', 'NFKC', or 'NFKD', got 'INVALID'
errors must be 'replace', 'ignore', or 'preserve', got 'unknown'
platform must be 'universal', 'windows', or 'posix', got 'bsd'
Invalid regex: <details from the regex engine>
```
